1. Assignment Overview 
This project implements the complete Minicrypt-to-MPC cryptographic reduction chain from scratch in 
Python 3.11, with no external cryptographic libraries. Every primitive must be built on top of your own 
prior implementation, traced from a One-Way Function (OWF) at the base all the way up to full 2-party 
Secure Multi-Party Computation. 
 
The 21 Programming Assignments (PA#0 – PA#20) span four parts: 
• Part I — Symmetric Cryptography (PA#1–6): OWF, PRG, PRF, Encryption, MACs 
• Part II — Hashing & Data Integrity (PA#7–10): Merkle-Damgård, DLP hash, Birthday attack, 
HMAC 
• Part III — Public-Key Cryptography (PA#11–17): DH, RSA, ElGamal, Signatures, CCA-PKC 
• Part IV — MPC (PA#18–20): Oblivious Transfer, Secure AND, Full 2-party computation 
• Part 0 — Minicrypt Clique Web Explorer (PA#0): React app built incrementally 
 
Critical Rules: 
• No external crypto libraries (no pycryptodome, cryptography, OpenSSL bindings, hashlib for 
HMAC, etc.). Only os.urandom and Python's built-in int (arbitrary precision) are permitted. 
• Every bidirectional reduction (marked ✓) must implement BOTH A=>B and B=>A directions. 
• Every PA must trace its entire dependency chain back to your own implementations. 
• You MAY use: os.urandom, Python int arithmetic, pytest, flask/fastapi (web bridge only).

5.3 Python-Specific Notes 
• Big integers: Python's built-in int is arbitrary precision — use directly. pow(base, exp, mod) is 
permitted for modular exponentiation as it's a built-in integer operation, not a crypto library. 
However, you must implement your own square-and-multiply to show understanding. 
• Randomness: use os.urandom(n) everywhere. Never use random.random(), secrets, or any other 
source. 
• AES-128: implement SubBytes (S-box), ShiftRows, MixColumns (GF(2^8) arithmetic), 
AddRoundKey, KeySchedule fully from scratch in aes128.py. 
• Byte manipulation: use Python bytes/bytearray; XOR via bytes(a ^ b for a, b in zip(x, y)). 
• No hashlib: not even for HMAC or SHA — PA#10 HMAC must use your own DLP hash from 
PA#8. 
• Web bridge: Flask or FastAPI is permitted ONLY to serve Python implementations as HTTP 
endpoints for the React app. All crypto logic stays in src/. 
 
6. Full Reduction Chain (How Everything Connects) 
Every arrow below must be traceable as actual Python function calls. Bidirectional arrows (<=>) require 
both directions implemented and demonstrated. 
 
Layer 1 — Minicrypt Core: 
• OWF (PA#1) <=> PRG (PA#1)  <=>  PRF (PA#2, GGM)  <=>  PRP (Luby-Rackoff) 
• PRF => MAC (PA#5)   |   PRF + MAC => CCA-Enc (PA#6)   |   PRF => CPA-Enc (PA#3) 
• CRHF (PA#7+8)  <=>  HMAC (PA#10)  <=>  MAC (PA#5)      [Birthday floor: PA#9] 
Layer 2 — Cryptomania: 
• OWF => DH (PA#11) => RSA (PA#12) => ElGamal (PA#16) 
• ElGamal/RSA => Signatures (PA#15) => CCA-PKC (PA#17) 
Layer 3 — MPC: 
• CCA-PKC (PA#17) => OT (PA#18) => Secure AND (PA#19) => All 2-Party MPC (PA#20) 
 
End-to-end call stack for one AND gate in PA#20: 
• secure_and(a, b)  [PA#19] 
◦ calls ot.receiver_step1(b) + ot.sender_step(pk0, pk1, 0, a)  [PA#18] 
◦   calls elgamal.encrypt(pk, m)  [PA#16] 
◦     calls dh.generate_group()  [PA#11] 
◦       calls prime_gen.gen_prime(bits)  [PA#13] 
◦         calls miller_rabin.is_prime(n)  [PA#13] 
◦           uses pow(a, d, n)  [Python built-in int — permitted] 
This full chain must be documentable in your README — no library shortcuts anywhere. 
 
7. Testing Strategy 
Per-PA pytest Files (tests/test_paN.py) 
• Correctness: decrypt(encrypt(m)) == m, verify(mac(m)) == True, etc. 
• Game simulations: IND-CPA, IND-CCA2, EUF-CMA — run 100 rounds, adversary advantage < 
0.1. 
• Attack demos: broken schemes must demonstrably fail when attacked (nonce reuse, deterministic 
RSA, etc.). 
• Bidirectional: for each PA marked ✓, test BOTH directions produce correct output. 
Shared / Integration Tests 
• PRG statistical: pytest collects p-values from NIST frequency, runs, serial tests — must all pass 
at p > 0.01. 
• Birthday empirical: 100 trials per n in {8,10,12,14,16} bits, ratio vs 2^(n/2) within 2×. 
• MPC lineage: assert that executing PA#20 Millionaire triggers PA#18 OT which triggers PA#16 
ElGamal (use mock/spy if needed). 
• Run all tests: pytest tests/ -v from project root.
